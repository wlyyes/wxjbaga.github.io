第一章 绪论

1 数据结构范畴
程序设计=算法+数据结构
2 数据结构的基本概念
（1）数据：数据不局限于具有字面量的数字，字符、图像、声音均可作为数据。
（2）数据元素：单个数据项可作为数据元素，同时多个字段组成的一条记录（sql描述）也可作为数据元素。
数据元素是怎么存储的？

（3）结构：指数据元素之间存在的关系。比如数组，链表，队列。
结构的分类：
<1>逻辑结构：表示数据之间逻辑关联的一种结构。
<2>存储结构：将数据之间的逻辑结构在计算机中表现出来。
（4）数据类型和抽象数据类型
数据类型：简单值（int，double，long……）或者复杂值（数组）和定义在此集合上的一组操作（+ - * /……）的总称。抽象数据类型ADT:在现有的数据类型上进行扩展（通过使用已有的数据类型），补充一组操作而构成的数据类型。我们可以这样理解ADT，ADT实际上是用户为了研究问题而构造出的一种数据类型。我们先来研究已有的数据类型int，int数据类型需要定义操作（+ - * /），但是在用户层面我们不知道它是怎么定义的，这叫做数据封装，类似于c++中封装的概念，同时我们需要为int数据类型设置接口以便用户来使用。这些工作已经提前被设置好，我们只需要直接去使用就行。当我们所研究的问题用已有的数据类型不方便或者无法实现时，我们就需要定义ADT。
抽象数据类型的定义格式：
ADT 抽象数据类型名
{
数据对象：（）
数据关系：（）
基本操作：（）
}
ADT的定义格式反映出数据类型的本质特征，实际上已有的数据类型（int、double、long）也具有这样的本质特征。
抽象数据类型=数据结构+操作。（数据结构依赖于现有的数据类型来实现）
（5）算法和算法的衡量
<1>有穷性：在有穷步骤，有限的时间内程序的功能可以实现。（步骤简洁，执行时间合理）
<2>确定性：算法中操作的实现是确定的，不是模糊的。例如一个数学问题：1+1=2。他就是确定的。
<3>可行性：基本操作是可以执行的，是恰当的，并且用户一看就明白。
<4>有输入：终端输入（一般是键盘输入）、内置输入？（例子：求1-100的素数，只键入100，循环执行100次就能输入1-100数字，1-100这些具体的数字实际上被内嵌了）。
<5>有输出：有执行的结果。
（6）算法的设计：
<1>正确的，中肯的：语法正确，没有逻辑错误，输入输出是匹配的，并且对于一些刁难的要求，给定一些合法但是复杂的输入，输出也是正确的。
<2>可读的：代码容易理解，可读性搞=高，读起来不晦涩。
<3>健壮性：对于非法的数据，程序能够对其进行合法的处理，不会因为数据非法导致程序崩溃。
<4>高效率和较低的存储量。
算法执行效率的衡量：我们一般采用事后分析的方法，算法的执行时间越短，算法的效率越高。算法的执行时间一般和五个因素有关：算法选用的策略、问题的规模、算法设计语言、机器代码的质量、指令的速度（后两项和计算机的软硬件有关系）。我们一般不用绝对的时间单位来衡量算法的效率，我们需要抛开计算机软硬件的因素。我们知道：算法=控制结构（顺序、分支、循环）+基本操作（对固有数据类型的操作），一般来说算法执行时间与算法中主要基本操作的执行次数成正比。为了客观衡量算法的小效率，我们引入时间复杂度的概念，用T（n）来表示，T（n）=O（f（n）），其中f（n）表示基本操作的执行次数，例O(n^2)表示数据量增大n倍时，算法的执行时间增大n的平方倍。

第二章 线性表

1 静态链表的简单描述
静态链表实际上是用数组来实现的，我们在将两个链表合成一个链表的时候，不需要重新建立新链表的空间，只需要更改链表结点之间的联系（通过指针实现），也可用一维数组来描述线性链表。
下面是以一个例题为背景的几个基本算法的伪码描述：
先建立表示集合A的静态链表S,然后在输入集合B的元素的同时查找S表，如果A表中存在和B表相同的元素，则将A表中重复元素删除，将B表中不重复元素插入A表。
<1>//线性表的静态单链表存储结构
#define MAXSIZE 1000//链表的最大长度
typedef struct
{
Elemtype data;//数据域
int cur;//下一个元素的地址
}component,SLinkList[maxsize];//这个结构体的名字是SLingList[maxsize]
<2>//初始化预先分配的空间
void InitSpace_SL(SLinkList &space)
{
//将一维数组space各元素链接成临时的链表，space[0].cur表示另一个元素的地址。
for(i=0;i<MAXSIZE-1;i++)
space[i].cur=i+1;
space[MAXSIZE-1].cur=0;
}
<3>//在备用链表中申请空间，分配节点
int Malloc_SL(SLinkList &space)
{
i=space[0].cur;
if(space[0].cur)
space[0].cur=space[i].cur;
return i;//返回申请节点的下标
}
<4>//将空闲节点还给临时链表
void Free_SL(SLinkList &space,int k)
{
space[k].cur=space[0].cur;
space[0].cur=k;
}
<5>//实现链表结合的算法
void difference(SLinklist &space,int &s)
{
//补充一点，space[0].cur表示备用空间的头指针，S表示新链表的头指针
InitSpace_SL(space);//初始化预先分配的空间
S=Malloc_SL(space);//生成新链表的头节点
r=S;
scanf(m,n);//读入A表和B表的大小
for(j=1;j<=m;++j)//建立A的链表
{
i=Malloc_SL(space);
scanf(space[i].data);//将A的元素输入
space[r].cur=i;//将该元素插入到表尾
r=i;//移动r使其指向最后的结点
}
space[r].cur=0;//设置尾节点的指针为空
for(j=1;j<=n;++j)//依次输入B的元素，若不在A表中，则插入，否则，在A表中删除
{
//k指向集合A中第一个结点，存储结点的地址
scanf(b);
p=S;
k=space[S].cur;
while(k!=space[r].cur&&space[k].data!=b)//k不指向尾结点，并且k所指结点的数据域不等于b值
{
//向后移动指针k
p=k;//p用来记忆k移动前的值
k=space[k].cur;//k进行移动
}
if(k==space[r].cur)//k指向了尾结点，表示A表中不存在该值，进行插入操作
{
i=Malloc_SL(space);
space[i].data=b;
space[i].cur=space[r].cur;
space[r].cur=i;
}
else//A表中存在该元素，进行删除操作
{
space[p].cur=space[k].cur;
Free_SL(space,k);
if(r==k)//如果删除的是尾结点，修改尾指针
r=p;//p是所删除结点的上一个结点
}
}
}
2 顺序存储结构和链式存储结构注意事项：
什么是线性结构呢？通俗来讲，每个结点对应一个结点，是一对一的关系。
顺序存储结构的存储一定是连续的，链式存储结构的存储不一定是连续的，可连续可不连续。顺序存储结构和链式存储结构都是线性结构的映像，例如数组或者链表都可以用来处理一对一的问题。另外在占用内存方面，链式存储结构比顺序存储结构占用的内存大，因为每个结点包含了数据域和指针域，占用了更多的空间。

第四章 串

1 概念：
串：由n个字符组成的有限序列，例如：’abcdef’。
子串： 例如串’abcd’属于串’abcdef’的子串
位置：字符在串中的序号称为位置，例如’abcdef’中字符a的位置为1，子串在串中的位置是子串第一个字符在串中的位置，例如’bcd’位置为2.
排序：abcd<abcf abc<acd 将两个字符串相比较，先从第一个字符开始比较，如果相等则比较下一个字符，注意字符之间比较的是他们的ASCII码值。
2 规律
（1）串的表示和实现
<1>定长顺序存储表示：
这种存储用的是连续的地址空间、存储单元，pascal中，第一个存储单元存放串的实际长度，在c语言中，以\0作为字符串的终结
<2>堆分配存储表示
这种存储用的也是连续的存储单元，在c语言中，存在“堆”自由存储区，动态分配函数：malloc()和free()函数来管理这个自由存储区。
<3>串的块链内存表示
这是一种用链表来存储串值的方法，用这种方式存储时，一个结点可以存放一个或者多个字符，选择一个结点放几个字符或者说我们选用几个结点来处理问题直接影响着串处理的效率。
引入这样一个定义：存储密度=（串值所占的存储位/实际分配的存储位）
当存储密度小，例如一个结点存放一个串值，运算起来方便。这话怎么讲？一个结点只存放了一个字符，应该是对单个字符的处理比较方便，比如说我想查找某个字符在字符串中出现的位置，结点存储一个字符时，可对这个串进行遍历操作，直到找到对应的字符。如果一个结点存放了多个字符，我如果想查找某个字符出现的位置，我需要遍历链表来找到哪个子串中存在这个字符，然后在符合条件的子串中再进行查找。显然结点存储一个字符时，进行这一操作比较方便。但是，存储密度小时也有弊端，结点太多，用到的指针太多，占用了过多的内存。
3 应用
<kmp算法> 与普通算法解决的问题一样，kmp算法解决的问题也是查找主串s中是否含有模式串，并且返回模式串的位置，但是kmp算法在执行效率上比普通算法高的多。
假设现在有文本串：aabaaaaf 模式串：aabaaf 要实现查找功能，我们可以发现模式串的前5个字符与文本串的前5个字符是对应的，模式串的f和文本串的b产生了冲突，若按照一般的解决办法，我们通常会这样做，让模式串整体向右移动一位，再与文本串对应匹配，如果还存在冲突，那么再次右移一位，直到模式串完全匹配文本串的子串。这种办法比较麻烦，kmp算法是这样做的，
我们先引入几个概念，前缀，后缀，前缀表，最长相等前后缀长度。
前缀：包含字符串首字母但不包含尾字母的字符串。后缀：包含字符串尾字母但不包含首字符的字符串。
前缀表：例如字符串aabaaf的前缀表是：0 1 0 1 2 0
当f不匹配时，f之前所有的字符肯定是匹配的，如果f前一个字符对应的前缀记录（前缀表中的一个值）不为0，为1、2、3，那么将模式串向右移动，让索引对应前缀记录的字符对应文本串中和f不匹配的字符，此时模式串前几个字符肯定和主串的子串是对应的。
想要实现kmp算法，next数组是很重要的，那么如何求这个next数组呢？伪代码如下：

void getnext(p,next)//p表示模式串，next表示所要求的next数组
{
j=0;//j是前缀末尾的下标，并且也是从j到i最长相等前后缀的长度
next[0]=0;//初始化next数组
for(int i=1;i<s.size();i++)
{
while(j>0&&s[i]=s[j])
{
j=next[j-1];
}
if(s[i]==s[j])
{
j++
}
next[i]=j;
}
}
