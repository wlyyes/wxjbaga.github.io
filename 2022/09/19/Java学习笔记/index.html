第三章 类和对象

类

<类的属性和方法>
类的属性和方法都是定义在类体中的。类的属性就是类的成员变量，如果不给成员变量赋初值，那么系统会给其赋默认值，int类型的变量的默认值是0，boolean类型变量的默认值是false，引用型变量的默认值是NULL。方法实际上就相当于c++语言中的成员函数，由类对象来调用，方法体中定义的变量 是局部变量，局部变量没有默认值，需要给他赋初值。

<类的构造方法>
构造方法也即构造函数，构造方法的方法名字和类名相同并且构造方法没有返回值，如果程序中没有关于该类构造方法的定义，那么系统会为该类生成一个默认的构造方法，如果该类给定了构造方法，那么程序不再为该类生成默认的构造方法，Java中的构造方法和c++中的默认构造函数是类似的，c++中如果没有显式给定该类默认构造函数，那么系统会为该类生成一个默认的构造函数。两种语言有共同的特点，结合起来学可能效果好一点。构造方法一般在创建对象时，用new操作符来调用。
（一段关于Java类的构造方法）
3.1.1
public class Student
{
//定义成员变量
String no;
String name;
int age;
int score1,score2;
//定义有参数的构造方法
public Student(String no,String name,int score1,int score2)
{
this.no=”no”;
this.name=”name”;
this.score=score1;
this.score2=score2;
}
//定义无参的构造方法
public Student(){}
int sum()
{
return score1+score2;
}
}
这个this是什么意思?与c++中的this是类似的，表示当前调用方法的对象。this一种特殊的用法，在一个构造方法中用this调用另一个构造方法。构造方法一般在创建对象时用new操作符调用，虽然Java和c++都是面向对象的高级语言，Java中的构造方法和c++中的默认构造函数有相通之处，但是也存在着差异，java中的构造方法可以重载而c++中的默认构造函数不能重载。
<java中构造方法的重载代码>
3.1.2
public class Rectangle

//成员属性

<类的继承>
在java中定义的每个类都是直接或者间接地继承java.lang.Object得到的
继承的关键字：extends
继承的格式：
class a(子类名) extends b(父类名)
{
//类体
}
3.1.3
public class Person
{
//定义成员变量
private String id;
private String name;
private int age;
//定义构造方法
public Person(String id，String name,int age)
{
this.id=id;
this.name=name;
this.age=age;
}
}
(一段关于继承的代码)

class student extends Person

//定义成员变量
String num;

int sum()
{
return score1+score1;
}
public student()

System.out.println(“子类的构造方法”);
}

关于继承，子类中只包含了和父类中不同的成员变量或者方法，有了继承之后，我们在设计类的时候，在父类中设置基本的成员变量和方法，
在派生类中根据需要扩展成员变量或者方法，当子类中的方法和父类中的方法重复时，子类中会自动覆盖父类中的方法。除extends关键字外，
在继承中还有一个关键字：Super。我们知道，当我们在子类中重写了父类的方法之后，在子类中无法访问父类被覆盖的方法，Super解决了这个问题，
用法为:super成员变量，super.成员方法。但是在调用的过程中有几点要求：通过super调用父类构造方法的代码需出现在子类构造方法的第一行

<类的访问修饰符>
类的访问控制修饰符有两种：public和默认。在一个源程序文件中可以声明多个类，但按照程序规范的要求，一般不建议在一个源程序文件中声明多个类。
在一个源程序文件中用public修饰的类只能有一个，并且类名必须与文件名相同。如果类用public修饰，则该类可以被其他所有的类访问如果类是默认的修饰符，
则该类只能被同一包中的类访问。
<成员的访问修饰符>
[public]：public所修饰的类成员可被所有的类访问，public指定的访问权限范围最大。
[protected]：类中由protected修饰的成员可以被该类本身、它的子类（包括同一个包中及不同包中的子类）及同一个包中的其他类访问。
[默认]：类中的成员为默认的访问权限时，成员可以被这个类本身和同一包中的类所访问，即包访问权限。
[private]：private所修饰的类成员只能被这个类本身访问。

<变量的访问修饰符>
[static]:
基本概念：static声明的变量是静态变量，在java类中，用static修饰成员变量之后，如果某一个对象修改了这个静态成员变量的值，
那么同一个类的所有对象都会受到影响。
应用：类变量的访问格式 类名.变量名 /对象名.变量名

扩展：static关键字也可以在声明方法时使用，用static关键字声明的方法叫做静态方法或者类方法，这样我们可以在不创建对象的情况下调用方法
应用：类方法调用格式 类名.方法名 /对象名.变量名

class CollegeStudent
{
static String schoolname;//定义静态变量
String name;
int age;

//定义有参的构造方法
public CollegeStudent()//构造方法的方法名字和类名相同并且没有返回值
{
this.name=name;//将形参的值赋给当前对象的成员变量
this.age=age;
}
//定义无参构造方法,java中类的构造方法必须加修饰符public吗——————————————————————————————
public CollegeStudent(){//方法体为空}
//定义类方法,也就是static修饰的静态方法
static void setSname()
{
schoolname=”清华”;
}
//定义类方法
static String getSname()
{
return schoolname;
}
}
public class StudentTest
{
publivc static void main(String [] args)
{
CollegeStudent z=new CollegeStudent(“coco”,18);//创建这个对象时，调用CollegeStudent类中的构造方法
z.schoolname=”齐鲁工业大学”;//通过对象引用类变量
CollegeStudent o=new CollegeStudent(“lili”,18);
CollegeStudent.schoolname=”青年政治学院”;//通过类名引用类变量
o.schoolname=”山东大学”;//改变对象所对应的成员变量的值
System.out.println(“coco的学校是：”+z.getSname());//通过对象调用类方法
System.out.println(“lili的学校是：”+CollegeStudent.getSname());//通过类名调用类方法
}
}

注意事项：非static声明的方法可以调用类方法或者类变量，类方法不能调用非静态的变量或者方法。
类方法是属于整个类的，而非静态变量或者方法只针对于某个对象才有意义，属于类时是空的，所以不可调用。

使用final修饰的类不能被继承，不可拥有子类。
使用final修饰的方法不能被子类重写。
使用final修饰的变量是常量，只能赋值一次，不可以修改。例如以下程序段：
final int num=1;
num =2;
//会产生编译错误,因为变量num只能被赋值一次，不可以修改

[abstract]
抽象方法：java中不包含方法体的方法称为抽象方法，抽象方法必须用abstract修饰
抽象类：类中如果有抽象方法,类就必须使用abstract修饰，该类被称为抽象类
抽象类的子类如果没有重写父类所有的抽象方法，那么子类仍然是抽象类。

abstract class Shape//表明该类是抽象类
{
public String color=”white”;//定义普通变量
public void sayColor()//抽象类中可以存在非抽象方法
{
System.out.println(“颜色是：”+color);
}
public abstract double getArea(){};//定义抽象方法
}
class CircleShape extends CircleShape//继承抽象类，实现父类的抽象方法
{
public double radius;//声明本身的属性，半径
public double getArea()//实现父类中的方法
{
return Math.PI * radius * radius;//类math不需要导入
}
}

包

如何访问另一个包中的共有类
第一种方法：在每个类名之前添加完整的包名，例如：a.b.c lei = new a.b.c();
第二种方法:使用import语句，例：import a.b.lei;/import a.b.*;

java编译器为所有程序自动引入包java.lang,所以所有的程序都可以访问包java.lang中的类
不再需要import语句引入
如果程序引入的两个包(import a.b.;,import t.b.)中存在相同的类t,那么在程序中使用这个重复的类时,就会报错。如果我们想使用包a的类t
解决办法：通过import引入具体的类 例import a.b.t;

接口

概念：接口是由常量和抽象方法组成的,可以理解为一种特殊的类。
接口定义的基本语法格式：
interface 接口名[extends 父类接口列表]
{
变量名=初值；//静态变量
返回值 方法名(参数表) throw[异常列表]；
//抽象方法
}
定义接口时，关键字interface之前的默认修饰符是public abstract
接口是多继承,类通过implments关键字实现接口，接口中的成员变量被系统自动增加public static final修饰符，
必须在定义时给其指定默认值，接口中所有的方法都是抽象的和公有的，在实现接口的类中重写抽象方法时，必须显式标明方法的
访问权限为public的。

interface CircleShape//定义接口
{
double PI=3.14159;//等价于public static final double PI=3.14159;
double area();//等价于public abstract double area();
}
class Circle implements CircleShape//实现接口
{
double radius;
Circle(double r)//有参构造方法
{
this.radius=r;
}
public double area()//实现抽象方法,显式标明方法的访问权限为public
{
return Math.PI * radius * radius;
}
}
public class Test
{
public static void main(String args[])//定义主函数
{
Circle c=new Circle(10);//创建对象的时候调用构造方法
System.out.println(“圆的面积为:”+c.area());
}
}

如果在定义类时，类既要实现接口又要继承父类，则语法格式如下例
先extends继承父类，再implements接口
interface Runner//定义接口
{
void fun();
}
interface Swimmer//定义接口
{
void swim();
}
abstract class Animal//定义抽象类
{
abstract public void eat();//抽象方法
}

class Dog extends Animal implements Runner,Swimmer//Dog类要实现这两个接口
{
String color;
Dog(String color)//有参构造方法
{
this.color=color;
}
public void run//实现接口抽象方法，必须加修饰符public
{
System.out.println(color+”dog is running”);
}
public void swim
{
System.out.println(color+”dog is swimming”);
}
public void eat()//实现父类抽象方法
{
System.out.println(color+”dog is eating”);
}
}
public class Test
{
public static void main(String args[])
{
Dog toni=new Dog(“white”);//
toni.run();
}
}
